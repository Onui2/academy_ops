# Academy Ops Hub 추가 기능 확장 PRD

- 작성일: `2026-04-27`
- 상태: `향후 구현 예정`

## 목적

현재 운영 시스템은 전산 장비, A/S, NAS, 기타 요청 중심으로 구성되어 있다.  
실제 운영 과정에서는 `결제선생 단말기` 및 `결제선생 관련 문의`도 지속적으로 발생하며, 요청을 유형별로 더 세분화해 관리할 필요가 있다.

또한 장비 요청 중 `태블릿`은 일반 PC/노트북과 다른 처리 흐름이 필요하다.  
사용자가 세부 부품을 직접 고르는 방식보다, 수량 중심으로 요청하고 이후 메일 요청 또는 담당자 직접 설정으로 처리되는 경우가 많기 때문이다.

이번 문서는 아래 기능을 향후 구현하기 위한 요구사항 정리 문서다.

1. `결제선생` 관련 요청 카테고리 추가
2. 사용자/관리자 화면에서 카테고리별 조회 기능 추가
3. 장비 요청 내 `태블릿` 전용 신청 흐름 분리

## 배경

현업에서는 다음과 같은 요청이 실제로 자주 발생한다.

1. 결제 단말기 설치 요청
2. 결제 단말기 장애 문의
3. 결제선생 계정/정산 문의
4. 결제선생 사용 관련 문의
5. 태블릿 신규 요청 및 세팅 요청

하지만 현재 시스템에서는 이를 명확하게 분리하지 못해, `기타` 또는 다른 카테고리로 우회 접수될 가능성이 있다.  
이로 인해 요청 분류 정확도, 처리 속도, 관리자 가시성이 떨어질 수 있다.

## 범위

### 포함 범위

1. 사용자 포털 카테고리에 `결제선생` 추가
2. 관리자 콘솔 요청 분류에 `결제선생` 추가
3. 사용자/관리자 모두 카테고리별 요청 조회 및 필터 기능 제공
4. `장비가 필요해요` 안에서 `태블릿`을 일반 장비 구성과 분리
5. 태블릿 요청 시 수량 기반 입력 방식 제공
6. 태블릿 요청의 처리 유형 구분 정보 저장

### 제외 범위

1. 실제 메일 자동 발송 기능
2. 외부 결제선생 API 연동
3. 태블릿 재고 관리
4. 결제선생 전용 자동화 워크플로우

## 핵심 요구사항

## 1. 결제선생 카테고리 추가

사용자 포털에 신규 카드형 카테고리 `결제선생`을 추가한다.

예상 요청 예시는 다음과 같다.

1. 결제 단말기 설치 요청
2. 결제 단말기 오류 접수
3. 결제선생 계정/정산 문의
4. 결제선생 사용법 문의

관리자 화면에서도 해당 요청이 `결제선생` 카테고리로 분류되어 표시되어야 한다.

## 2. 카테고리별 조회 기능

### 사용자 포털

사용자는 본인이 접수한 요청을 아래 기준으로 조회할 수 있어야 한다.

1. 전체
2. 전산 장비
3. A/S
4. NAS
5. 결제선생
6. 기타

### 관리자 콘솔

관리자는 요청 큐에서 카테고리 기준으로 목록을 필터링할 수 있어야 한다.

목적은 다음과 같다.

1. 특정 유형 요청만 빠르게 확인
2. 결제선생/장비/A/S 요청 분리 운영
3. 업무 유형별 처리 우선순위 판단 보조

## 3. 태블릿 요청 분리

`장비가 필요해요` 카테고리 안에서 `태블릿`은 일반 노트북/데스크톱처럼 부품 조합 방식으로 처리하지 않는다.

예상 흐름:

1. 사용자가 `장비가 필요해요` 선택
2. 장비 유형 중 `태블릿` 선택
3. 태블릿 전용 입력 폼 노출

태블릿 요청 전용 입력 항목 예시:

1. 학원/지점
2. 태블릿 수량
3. 사용 목적
4. 요청 상세 메모
5. 긴급 여부
6. 첨부 파일

핵심 원칙은 `태블릿은 몇 대 필요한지 중심으로 요청한다`는 점이다.

## 4. 태블릿 처리 방식 구분

태블릿 요청은 내부적으로 두 가지 성격이 있다.

1. 신청 후 메일로 요청하는 건
2. 담당자가 직접 보고 설정해주는 건

따라서 태블릿 요청에는 관리자 확인용 구분 정보가 필요하다.

예시 필드:

1. 처리 방식: `메일 요청` / `직접 세팅`
2. 추가 메모
3. 전달 또는 설치 필요 여부

이 정보는 최소한 관리자 화면에서 구분 가능해야 한다.

## 사용자 스토리

1. 사용자로서, 결제 단말기 관련 문의를 `기타`가 아닌 `결제선생`으로 정확하게 접수하고 싶다.
2. 사용자로서, 내가 접수한 요청을 카테고리별로 나눠서 보고 싶다.
3. 사용자로서, 태블릿 요청 시 복잡한 부품 선택 없이 수량 중심으로 간단히 접수하고 싶다.
4. 관리자로서, 결제선생 관련 요청만 따로 모아서 보고 싶다.
5. 관리자로서, 태블릿 요청이 일반 장비 요청과 다르다는 점을 한눈에 구분하고 싶다.

## 기능 상세

### 사용자 포털

1. 카테고리 카드에 `결제선생` 추가
2. 내 접수 현황 영역에 카테고리 필터 추가
3. 장비 요청 내 `태블릿` 분기 추가
4. 태블릿 선택 시 기존 커스텀 PC 사양 UI 비노출
5. 태블릿 요청용 수량 기반 폼 제공

### 관리자 콘솔

1. 요청 목록에 카테고리 필터 추가 또는 강화
2. `결제선생` 요청 전용 분류 표시
3. 태블릿 요청에 `태블릿` 세부유형 표시
4. 태블릿 요청 상세에서 처리 방식 확인 가능

## 데이터 모델 변경 제안

기존 요청 데이터에 아래 필드 추가를 검토한다.

1. `category`
   예: `equipment`, `as`, `nas`, `payment_teacher`, `other`
2. `subtype`
   예: `desktop`, `laptop`, `tablet`, `terminal_issue`, `settlement_inquiry`
3. `handlingType`
   예: `email_request`, `direct_setup`
4. `quantity`
   태블릿 요청용 수량
5. `requestChannelNote`
   처리 참고 메모

장기적으로는 `module` 단일 구분보다 `category + module + subtype` 구조가 더 적합하다.

## 수용 기준

1. 사용자 포털에 `결제선생` 카드가 추가된다.
2. 결제선생 요청이 관리자 화면에서 같은 카테고리로 식별된다.
3. 사용자와 관리자는 요청 목록을 카테고리별로 필터링할 수 있다.
4. `장비가 필요해요` 안에서 `태블릿`을 별도 유형으로 선택할 수 있다.
5. 태블릿 선택 시 기존 PC 부품 구성 UI가 나타나지 않는다.
6. 태블릿 요청은 수량 기반으로 접수된다.
7. 관리자 화면에서 태블릿 요청과 일반 장비 요청을 구분할 수 있다.

## 오픈 이슈

1. 결제선생 카테고리 명칭을 그대로 사용할지, 더 일반적인 결제/단말기 명칭으로 확장할지
2. 태블릿의 처리 방식을 사용자가 직접 선택할지, 관리자가 후처리로 지정할지
3. 카테고리 필터 UI를 탭으로 둘지, 드롭다운으로 둘지
4. 기존 `module` 중심 구조를 얼마나 유지할지
5. 결제선생 세부 유형까지 1차 범위에 포함할지
